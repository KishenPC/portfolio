## Overview

A living collection of system design explorations. Each entry models a real distributed systems problem, walks through constraints and tradeoffs, and provides a reference implementation sketch.

## Current Entries

- **Rate Limiter**: Sliding window, token bucket, and distributed rate limiting strategies.
- **Message Queue**: At-least-once delivery, dead letter handling, and consumer group semantics.
- **API Gateway**: Routing, authentication, rate limiting, and request transformation.
- **Event Sourcing**: Event store design, projection strategies, and CQRS patterns.

## Approach

Each design doc follows a consistent structure: problem statement, constraints, candidate approaches, tradeoff analysis, chosen design, and implementation notes. The goal is to make architectural reasoning visible and transferable across projects.
